Gortex × Understand-Anything

Самое главное в трёх частях: как работает gortex, как работает UA, и как они работают вместе.

① gortex · структура ② UA · смысл ③ мост · вместе
① gortex · суть

Типизированный граф кода

  • tree-sitter парсит код → граф знаний: узлы = символы, рёбра = calls, implements, reads_env, writes…
  • Детерминированно: «все вызывающие X», «что сломается» — точный ответ, не вероятностный.
  • Живой демон: граф в синхроне с кодом, инкрементальные апдейты — не снимок.
RAG угадывает по похожести. Gortex знает из структуры — и честно говорит, когда не разрешил.
① gortex · возможности

Не только чтение

Навигация / анализ

  • find_usages, вызывающие, реализаторы интерфейса.
  • Анализ влияния: что сломается от смены сигнатуры.
  • Capability-рёбра: «что читает секрет», «что шеллит наружу» — за один хоп.

Правки / интеграция

  • edit_symbol, rename_symbol, verify_change.
  • MCP-сервер (100+ tools) + HTTP /v1 + CLI.
  • Токен-эффективно: сигнатуры без тел (~30% токенов).
① gortex · поиск

Гибрид + петля обучения

  • BM25 (лексика) + эмбеддинг-мини-модель (вшитый GloVe-50d / опц. MiniLM-384 / jina_code) — семантический канал.
  • 19-сигнальный рерэнк: bm25, semantic, fan_in, hits, community, churn, recency, definition_bias, … (веса ∑≈6.0).
  • Relevance-feedback (вес 0.5): агент метит контекст useful/not_needed/missing → полезное всплывает, лишнее тонет, упущенное авто-включается. Это adaptive reranking, не дообучение модели.
Семантика — как подсказка. Структура — как истина. Это не полноценный RAG: эмбеддинги лишь вход в граф.
② UA · суть

Граф знаний для людей

  • MIT-плагин для AI-ассистентов (Claude Code · Codex · Cursor · …).
  • Превращает репо в интерактивный граф знаний: plain-English саммари, бизнес-домены, гайд-туры, persona-views.
  • Артефакт knowledge-graph.json + React-дашборд для онбординга.
② UA · как работает

Мультиагентный конвейер

scan (языки/файлы) → batch → file-analyzer (tree-sitter структура + LLM семантика) → merge → architecture (слои) → tour → review → knowledge-graph.json + React-дашборд
Структуру даёт tree-sitter, смысл — LLM-агенты. Ровно сюда врезается gortex.
③ вместе · зачем

Машина + человек

gortex структура

  • Детерминированный граф, dataflow, cross-repo, 256 языков.
  • Для агентов и инструментов.

UA смысл

  • Нарратив, домены, туры, дашборд.
  • Для людей (онбординг).
Слабое место одного — сильная сторона другого. Вместе: один граф, структура + смысл.
③ вместе · мост

Четыре стыка

  • L1 — gortex export understand → формат UA (understand-anything@1), детерминированно.
  • L1.1 — тот же экспорт как MCP-tool + HTTP /v1.
  • L2 — UA берёт структуру из gortex вместо своего tree-sitter; LLM только добавляет смысл.
  • L3 — UA пишет смысл обратно в gortex (annotate_nodes → Node.Meta), идемпотентно, без правки структуры.
Аддитивно и с graceful degradation: без gortex UA работает по-старому.
