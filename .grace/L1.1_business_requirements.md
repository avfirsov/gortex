# Business Requirements — L1.1: export_understand over MCP + HTTP

> GRACE task spec. Target: /mnt/d/code/gortex (branch gortex-ua-bridge). Builds on committed L1 (commit d5112e1).
> Self-contained: the exact integration points (from a codebase survey) are embedded below.

## 1. GOAL
Expose the existing, tested `exporter.WriteUnderstandAnything` as an MCP tool `export_understand`. Because the daemon's HTTP `/v1/tools/{name}` dispatches against the same MCP registry, registering the MCP tool **automatically** exposes `POST /v1/tools/export_understand` — no separate HTTP code.

## 2. WHY
L1 delivered the library + CLI. Agents and the UA dashboard reach gortex over MCP/HTTP, not the CLI. This is the thin wrapper that makes the L1 exporter reachable by the same surface as the other 100+ tools. It mirrors the existing `export_graph` tool exactly.

## 3. INTEGRATION POINTS (authoritative — from codebase survey)
- Reference tool to mirror: `internal/mcp/tools_export.go` — `registerExportTools()` (lines 18-35) + `handleExportGraph()` (lines 37-133) + `exportByFormat()` (lines 125-133).
- Registration API: `s.addTool(mcp.NewTool("name", mcp.WithDescription(...), mcp.WithString(...), mcp.WithBoolean(...)), s.handleX)`.
- Register call site: `internal/mcp/server.go:1056` (`s.registerExportTools()` inside `NewServer`) — add `s.registerUnderstandTools()` next to it.
- Handler signature: `func (s *Server) handleExportUnderstand(ctx context.Context, req mcp.CallToolRequest) (*mcp.CallToolResult, error)`.
- Graph access: `g := s.graph` (type `graph.Store`), passable directly to `WriteUnderstandAnything(w io.Writer, g graph.Store, opts exporter.UAOptions) (exporter.Stats, error)`.
- Arg helpers (already in the mcp package): `req.GetArguments()`, `stringArg`, `stringArgOrDefault`, `boolArgValue`, `splitCSVArg`.
- Output convention (from handleExportGraph): when `output_path` is set, write the file and return a stats summary; otherwise return the rendered JSON inline via `mcp.NewToolResultText(...)`.
- HTTP: `internal/server/handler.go:141` registers `POST /v1/tools/`; `handleToolCall` (369-424) looks the tool up via `h.mcpServer.GetTool(name)` — the SAME server instance, so the new tool is auto-exposed. No HTTP code to add.

## 4. SCOPE (feature slice)
IN:
- New file `internal/mcp/tools_understand.go`: `registerUnderstandTools()` + `handleExportUnderstand()`.
- Register it in `NewServer` (server.go) next to `registerExportTools()`.
- Correctness fix in `internal/exporter/understand.go`: remove `omitempty` on `UAProject.GitCommitHash` (the UA schema's `ProjectMetaSchema.gitCommitHash` is `z.string()` REQUIRED; an empty string is valid but an OMITTED field fails validation). The Action layer must always emit the field, even when no commit is known.
- Tests in `internal/mcp/` for the handler.
OUT: no changes to CLI, to existing export_graph, or to the L1 pure core (besides the one struct-tag fix).

## 5. TOOL CONTRACT
Name: `export_understand`. Description: "Export the code graph as an Understand-Anything knowledge graph (understand-anything@1, or generic@1). With output_path the daemon writes the file; otherwise returned inline."
Params (all optional):
- `granularity` string: "slim" (default) | "full".
- `generic` bool: emit generic@1 instead of understand-anything@1.
- `repo` string: filter to one repo prefix.
- `project_name` string: override UA project name.
- `output_path` string: absolute path to write; omit to return inline.
Handler: build `exporter.UAOptions{ Options: {Repo}, Granularity, Generic, ProjectName, AnalyzedAt: <now RFC3339>, GitCommit: <best-effort> }`, call `WriteUnderstandAnything`. AnalyzedAt = current time (Action layer). GitCommit = best-effort: if a single `repo` filter resolves to a tracked repo with a known commit, use it; else "" (and the struct-tag fix keeps the envelope valid).

## 6. ACCEPTANCE CRITERIA

- [ ] AC3: output passes UA `validateGraph` with zero dropped/fatal — **including the empty-commit case** (the gitCommitHash struct-tag fix). Reuse the L1 node harness (skip-guarded) + the always-on Go sanity oracle.
- [ ] AC4: `generic` param yields generic@1.
- [ ] AC5: gates — `go build ./...`, `go vet ./internal/mcp/ ./internal/exporter/`, `gofmt -l` clean; existing export_graph and L1 tests still pass (no regression); the full exporter package + mcp package tests green.
